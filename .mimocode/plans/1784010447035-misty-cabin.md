## Fix H2: Non-Project Issues Visibility for Regular Users
**File:** `backend/src/modules/projects/projects.service.ts:432-452`
**Current:** USER role filter only shows issues in projects they're members of. Issues WITHOUT a projectId are invisible.
**Fix:** Add OR clause to include issues where user is directly involved (raised or assigned)
**Business Logic Preserved:** Users still can't see project-scoped issues they're not members of, but can see their own non-project issues

```typescript
// Change lines 445-451 from:
return {
  project: {
    users: {
      some: { userId: actor.userId },
    },
  },
};

// To:
return {
  OR: [
    // Issues in projects the user is a member of
    {
      project: {
        users: {
          some: { userId: actor.userId },
        },
      },
    },
    // Issues without a project where user is directly involved
    {
      projectId: null,
      OR: [
        { raisedById: actor.userId },
        { assignedToUserId: actor.userId },
      ],
    },
  ],
};
```

## Fix H6: Deadline Warning Calculation Using Assignment Time
**File:** `backend/src/modules/notifications/notifications.service.ts:196-228`
**Current:** Warning calculated from `createdAt`. If issue sits unassigned for weeks then gets assigned, warning fires immediately.
**Fix:** Use `updatedAt` as a proxy for last assignment time (simpler than adding new field)
**Business Logic Preserved:** Warning still fires at 80% threshold, just calculated from more recent timestamp

```typescript
// Change lines 197-203 from:
const createdAt = issue.createdAt;
const totalDuration = deadline.getTime() - createdAt.getTime();

if (totalDuration <= 0) continue;

const elapsed = now.getTime() - createdAt.getTime();
const pct = (elapsed / totalDuration) * 100;

// To:
const createdAt = issue.createdAt;
const totalDuration = deadline.getTime() - createdAt.getTime();

if (totalDuration <= 0) continue;

// Use updatedAt as proxy for last meaningful activity (assignment, status change)
const lastActivity = issue.updatedAt || createdAt;
const elapsed = now.getTime() - lastActivity.getTime();
const pct = (elapsed / totalDuration) * 100;
```

**Note:** This requires adding `updatedAt` to the select query on line 171.

## Fix H7: Orphaned IssueAssignee Records on User Permanent Delete
**File:** `backend/src/modules/users/users.service.ts:255-281`
**Current:** `permanentRemove` deletes `issueAssignee` records but doesn't log the removal in activity logs
**Fix:** Add activity log entry before deleting assignee records
**Business Logic Preserved:** Same cleanup, just with better audit trail

```typescript
// After line 268 (projectUser.deleteMany), add:
// Log assignee removals for audit trail
const assigneeIssues = await tx.issueAssignee.findMany({
  where: { userId: id },
  select: { issueId: true },
});
for (const ai of assigneeIssues) {
  await tx.activityLog.create({
    data: {
      issueId: ai.issueId,
      userId: id,
      action: 'ASSIGNEE_REMOVED',
      oldValue: JSON.stringify({ userId: id }),
      newValue: null,
    },
  });
}
```
